# 配付物管理

## この機能でできること

進学相談会の各会場で来場者に配る資料や備品を一元管理するための画面です。
顧客（CL）から預かった資料や、自社で用意する配布物について、**どの会場に何部ずつ送るか**を管理できます。

主な機能：

- **品目の登録・管理** — CL名・資料名・セクション（封入/単独配布/置き/備品）を登録
- **品目の並び順をドラッグで変更** — セクション内で表示順を自由に並べ替え
- **会場ごとの部数管理** — 品目ごとに各会場への倉庫部数・事業課部数を入力
- **持ち回り管理** — 同じルート内で次の会場に持ち回る荷物をチェックボックスで管理
- **作業済チェック** — ルート別タブで品目ごとに作業完了をチェック
- **ルート別の一覧表示・印刷** — 物流ルート単位で一覧表示し、現場で使える形式で印刷
- **xlsx取込** — 顧客からの申込書（Excel）を取り込んで品目と部数を自動登録

## 画面の開き方

1. [ポータル](../portal.html) にアクセスしてパスワードを入力
2. メニューの「配付物管理」をクリック

---

## 画面構成

画面は5つのタブで構成されています。

| タブ | 用途 |
|---|---|
| **品目一覧** | 登録済みの品目を一覧表示。品目の登録・編集・削除・並べ替えを行うメインの画面。行クリックで**品目詳細モーダル**が開きます |
| **ルート別** | 物流ルートを選んで、ルートに含まれる会場と品目の部数を横展開で確認・印刷 |
| **会場別** | 期全体の全企画×封入品目のクロス表。どの会場にどの封入品目の申込があるかを俯瞰把握 |
| **更新履歴** | 配布物マスタ・配布先明細の直近30件の更新履歴（更新日の新しい順） |
| **xlsx取込** | 顧客からの申込書（Excel）をアップロードして品目を一括登録 |

## 品目一覧タブ

### 基本操作

1. 画面を開くと、選択中の期に登録されている品目が一覧表示されます
2. **期**を変更すると、その期の品目が自動的に読み込まれます

### 並び順

品目は以下のルールで自動的にソートされます：

1. **セクション順** — 封入 → 単独配布 → 置き → 備品
2. **セクション内の並び順** — ドラッグで設定した順番（後述）
3. **CL名順** — 並び順未設定時のフォールバック

各セクションの境界には見出し行が表示されるので、グループが視覚的にわかります。

### ドラッグで並べ替え

各行の左端にある **☰ ドラッグハンドル**をつかんで上下にドラッグすると、品目の表示順を入れ替えられます。

- **同じセクション内でのみ**並べ替え可能（封入の品目を備品に移動することはできません）
- 並べ替え後はサーバーに自動保存されます
- セクション自体を変更したい場合は、セクション列のセレクトボックスを使ってください（変更後はそのセクションの末尾に自動移動します）

### フィルタ

| フィルタ | 操作 |
|---|---|
| **期** | 前期・後期を切り替え |
| **セクション** | 封入 / 単独配布 / 置き / 備品 で絞り込み |
| **検索** | CL名・資料名のテキスト検索 |

### 表示される列

| 列 | 内容 | 編集 |
|---|---|---|
| ☰ | ドラッグハンドル（並べ替え用） | - |
| セクション | 封入・単独配布・置き・備品（セレクトボックスで変更可能） | 可 |
| CL名 | 顧客名・提供元名 | 可 |
| 資料名 | 品目の名称 | 可 |
| 外見 | 物理的な外観の説明（A4、白封筒 等） | 可 |
| 倉庫 | 倉庫向け部数の合計 | - |
| 事業課 | 事業課向け部数の合計 | - |
| 合計 | 倉庫＋事業課の総計 | - |
| 会場数 | 部数が入力されている会場の数 | - |
| 入庫予定 | 倉庫への入庫予定日 | 可 |
| 入庫日 | 実際に入庫した日 | 可 |
| 備考 | メモ欄（内容に応じて高さが自動伸縮） | 可 |
| 更新日 | 最後に情報を更新した日（自動記録） | - |
| x | 品目の削除ボタン | - |

> 編集可能な列は、セルを直接クリックして入力できます。入力欄から離れると自動的に保存されます。
>
> CL名・資料名を変更すると、その品目に紐づく全ての明細データも連動して更新されます。
>
> **備考欄**は内容に合わせて自動的に高さが伸びるので、長いメモも見やすく入力できます。

### 品目を新しく登録する

1. 「+ 品目登録」ボタンをクリック
2. 期（自動入力）、セクション、CL名、資料名、外見、備考を入力
3. 「登録」をクリック

登録すると、封入セクションの場合は企画設定の封入数に基づいて各会場の部数が自動設定されます。
それ以外のセクションでは、品目だけ登録され、部数は品目詳細パネルで個別に入力します。

新しく登録された品目は、そのセクションの末尾に追加されます。

### 品目を削除する

品目の行の右端にある「x」ボタンをクリックすると、確認ダイアログが表示されます。
削除すると、その品目に紐づく全ての会場別明細データも一緒に削除されます。

## 品目詳細モーダル（会場別の部数入力）

品目一覧の行をクリックすると、**品目詳細モーダル**がオーバーレイで開きます。
ここで各会場への部数や入庫情報を入力・確認できます（以前はインラインパネル展開でしたが、画面の見やすさ向上のためモーダル化しました）。

### 表示される列

| 列 | 内容 | 編集 |
|---|---|---|
| ルート | 物流ルート名 | - |
| 開催都市 | 会場の都市名 | - |
| 開催日 | 会場の開催日（曜日付き） | - |
| 倉庫 | 倉庫に送付する部数（数字またはテキスト） | 可 |
| 事業課 | 事業課に送付する部数（数字またはテキスト） | 可 |
| 入庫予定 | 入庫予定日 | 可 |
| 入庫日 | 実際の入庫日 | 可 |
| 入庫備考 | 入庫に関するメモ | 可 |
| 持ち回り | 持ち回りフラグ（チェックボックス） | 可 |
| リミット | 倉庫への発送期限 | - |
| 管轄 | 担当支社（東京本社・札幌支社 等） | - |
| 備考 | メモ欄 | 可 |

> 全ての会場（企画設定に登録されている全会場）が表示されます。部数が入力されていない会場にも入力可能で、入力すると自動的に明細データが作成されます。

### 倉庫部数・事業課部数のテキスト入力

倉庫・事業課の部数欄は**数字以外のテキストも入力できます**。
たとえば看板などで「合同」「単独」のような状態を記録したい場合に便利です。

- 数字を入れた場合 → 通常通り部数として集計されます
- テキスト（「合同」「単独」等）を入れた場合 → ルート別タブにそのまま表示されます。合計欄には加算されません

### 並び順

会場は以下の順番で表示されます：

1. **管轄順** — 東京本社 → 札幌支社 → 仙台支社 → 大阪支社 → 福岡支社
2. **ルート順** — 同じ管轄内では、ルート内の最も早い開催日が先のルートから
3. **開催日順** — 同じルート内では開催日が早い順

ルートごとに背景色が交互に変わるので、ルートの境界が視覚的にわかります。

### 持ち回りについて

同じルート内で、ある会場の資料をそのまま次の会場に持ち回る場合があります。
持ち回りに該当する会場で**「持ち回り」列のチェックボックスをON**にすると、ルート別タブでその会場のセルに「→」が自動表示されます。

- チェーンの先頭の会場には倉庫部数を入力し、後続のチェック対象会場は0または空欄でOK
- ルート別タブを見ると、持ち回り会場には「→」が表示されるので、現場で「ここは前の会場から持ち回し」と一目でわかります

> **注意**：年次更新で会場順が入れ替わった場合、チェックボックスの位置を新しい会場順に合わせて入れ直す必要があります。

## ルート別タブ

物流ルート単位で品目の配分を確認する画面です。倉庫で発送作業を行う際の確認用に使い、印刷もできます。

### 操作

1. 「ルート」セレクトボックスからルートを選択
2. 「抽出」ボタンをクリック
3. そのルートに含まれる会場が列として横に展開され、品目ごとの倉庫部数が一覧表示されます

### ルートセレクトの並び順

- 管轄ごとにグループ化（東京本社 → 札幌支社 → 仙台支社 → 大阪支社 → 福岡支社）
- 同じ管轄内ではルートの最初の開催日が早い順

### 品目の並び順

ルート別タブの品目順は**品目一覧タブと完全に同じ**です：

1. セクション順（封入 → 単独配布 → 置き → 備品）
2. セクション内のドラッグ並び順
3. CL名順

各セクションの境界には見出し行（左寄せ）が表示されます。

### 表示される列

| 列 | 内容 |
|---|---|
| # | 行番号 |
| CL名 | 顧客名 |
| 資料名 | 品目名 |
| 各会場 | 開催日・都市名・封入数。セルには倉庫部数（または「→」「合同」等のテキスト）を表示 |
| 合計 | 品目の全会場の合計部数（数字のみ加算） |
| 備考 | 品目のメモ |
| 済 | 作業完了チェック（チェックボックス） |
| 印刷除外 | チェックすると印刷時にその行が非表示になります（画面上は表示されたまま） |

> **封入・単独配布・置き**セクションの品目は、倉庫部数が全会場とも0でテキストも持ち回りもない場合は表示されません。
> **備品**セクションの品目は、部数が入力されていなくてもマスタに登録があれば常に表示されます。
> フィルタバーの「**データありのみ**」チェックをONにすると、備品の空行を非表示にできます。

### 作業済チェック

「済」列のチェックボックスで品目ごとに作業完了を記録できます。

- セクションを問わず**全ての品目**にチェックを付けられます
- チェックは品目の全会場の明細にまとめて反映されます
- サマリーバーに「作業済: N/M」と現在の進捗が表示されます

### 持ち回り会場の表示

品目詳細で持ち回りチェックを付けた会場は、ルート別タブのセルに **→** が自動表示されます。
これにより「この会場は前の会場から荷物を持ち回す」ことが現場で一目でわかります。

### 印刷

「印刷」ボタンをクリックすると、ブラウザの印刷ダイアログが開きます。
印刷時は以下の調整が自動で適用されます：

- 上部に「**配付物管理**」タイトル + 期/ルート名 + 出力日が表示
- ヘッダー・タブ・フィルタバー・ボタンなどの操作系UIは非表示
- 背景色・バッジ・チェックボックスも印刷対応

### サマリー

画面上部に品目数・会場数・合計部数・作業済件数が表示されます。

## 会場別タブ

期全体の全企画（会場）×封入セクション品目のクロス表です。ルートごとに絞り込まなくても、**どの会場にどの封入品目の申込があるかを一覧で俯瞰**できます。

### 操作

1. 期を選択して「抽出」ボタンをクリック
2. タブを切り替えると初回は自動で読み込みます

### 表示内容

| 要素 | 内容 |
|---|---|
| **行** | 当該期の全企画。管轄順（東京本社→札幌→仙台→大阪→福岡）→ ルート → 日付順 |
| **列** | 封入セクション品目（CL名+資料名を2行ヘッダーで表示） |
| **セル** | 部数あり＝緑背景＋数字 / 部数なし＝空白 |
| **品目数** | 各会場の封入品目件数 |
| **合計** | 各会場の封入部数合計 |

### グループ区切り

- **管轄区切り** — 濃紺ヘッダーで東京本社、札幌支社…と区切ります
- **ルート区切り** — 薄い青ヘッダーでルート名を表示

### 注意事項

- **封入セクションのみ**が対象です（単独配布・置き・備品は含まれません）
- **CL名が栄美通信（社内品）の品目は、東京本社管轄の会場についてのみ包含**されます。札幌・仙台・大阪・福岡の支社管轄会場では従来通り除外（2026-04-27変更）
- 部数が全会場とも0の品目は列に表示されません
- サマリーバーに企画数・封入品目数・合計部数が表示されます
- 印刷にも対応しています

> **「ルート別」と品目数が違う件**：ルート別は全セクション（封入・単独配布・置き・備品）を含み栄美通信も全会場で含まれるのに対し、会場別は封入セクション限定＋栄美通信は本社のみ＋部数>0のみ という3つの絞り込みが入るため、品目数が異なります。これは仕様です。

### こんなときに使います

- 「この期の封入品目がどの会場に入っているか」を一覧で確認したい
- 「申込がまだ来ていない会場はどこか」を素早く把握したい
- 現場の倉庫スタッフに全体感を共有したい

## 更新履歴タブ

配布物マスタ・配布先明細の **直近30件の更新履歴** を新しい順に表示します。

### 表示される情報

| 列 | 内容 |
|---|---|
| 更新日 | 情報が変更された日付 |
| シート | 配布物マスタ / 配布先明細 |
| CL名 | 対象品目のCL名 |
| 資料名 | 対象品目の資料名 |
| 会場 | 配布先明細の場合は対象会場（マスタの場合は空白） |
| 変更内容 | 何の列が変わったか |

### こんなときに使います

- 「最近誰がどの品目を変更したか」を確認したい
- 「この品目はいつ更新されたか」を素早く把握したい
- 編集ミスがあったときの原因追跡

## xlsx取込タブ

顧客からの配付物申込書（Excel）を取り込み、品目と会場別部数を自動登録します。

### 対応するExcelフォーマット

申込書は以下の構造を想定しています：

- **各シート = 1品目**（No.1, No.2, No.3...）
- シート上部に「貴学名」「配付資料名」「納品予定日」「入庫日」「備考」
- シート下部に地区ごとの会場リスト（会場名・開催日・配付希望部数・備考）

### 取込手順

1. **xlsx取込タブ**を選択
2. **期**を選択（品目マスタにこの期が記録されます）
3. xlsxファイルをドラッグ＆ドロップ、またはクリックして選択
4. 「取込実行」をクリック
5. 結果メッセージが表示されます：
   - **追加件数** — 新規に登録された品目の数
   - **スキップ件数** — 既に同名の品目が存在するためスキップされた数
   - **未マッチ都市** — 企画設定に該当する会場が見つからなかった都市名

### 取込の仕組み

- Excelの会場名と開催日を、スプレッドシートの**企画設定**の開催都市・開催日と照合してマッチします
- マッチした会場には、申込書の部数が**倉庫部数**として自動入力されます
- 部数が0の会場は明細データを作成しません（軽量化のため）
- 同じCL名・資料名の品目が既に存在する場合はスキップされます
- 取り込んだ品目はそのセクションの末尾（並び順）に追加されます

### 未マッチ都市について

以下のケースでは会場がマッチしません（正常な動作です）：

- 開催日が「未定」「9月開催予定」など具体的な日付が入っていない場合
- 企画設定にまだ日程が登録されていない会場
- 会場名の表記が企画設定と異なる場合

未マッチの会場は、取込後に品目詳細パネルから手動で部数を入力できます。

## セクションの種類

品目は4つのセクションに分類されます。

| セクション | 説明 |
|---|---|
| **封入** | ファッションバッグなどに封入して配布するもの。登録時に企画設定の封入数が自動入力されます |
| **単独配布** | 封入せず単独で来場者に手渡しするもの |
| **置き** | 会場の指定場所に積み置きして配布するもの |
| **備品** | 会場運営に関わる備品（ポール、テーブルクロス 等） |

## レスポンシブ対応

スマートフォン・タブレットからもアクセスできます。

- 画面幅768px以下では、フォントサイズ・カラム幅・ボタンサイズが自動的に縮小されます
- テーブルは横スワイプで全列を確認できます
- 列数の多い品目一覧は、iPhoneサイズだとやや窮屈ですが、現場確認用には十分使えます

## スプレッドシートとの関係

この画面のデータは、スプレッドシートの以下のシートに保存されます。

| シート名 | 内容 |
|---|---|
| **配布物マスタ** | 品目の基本情報（CL名・資料名・セクション・入庫日・並び順 等） |
| **配布先明細** | 品目×会場の部数データ（倉庫部数・事業課部数・入庫情報・持ち回り・封入済 等） |

> シートを直接編集することもできますが、通常はこの画面から操作してください。

## よくある質問

**Q: 品目の並び順を変えたい**
A: 品目一覧タブで、行の左端にある ☰ ハンドルを上下にドラッグしてください。同じセクション内で並べ替えられ、自動保存されます。

**Q: セクションを変えたら品目がどこかに行ってしまった**
A: セクションを変更すると、新しいセクションの末尾に自動移動します。新しいセクションのグループまでスクロールしてください。

**Q: 倉庫部数に「合同」と入れたい**
A: 倉庫部数欄は数字以外のテキストも入力できます。「合同」「単独」「未定」など自由に入力してOKです。ルート別タブにもそのまま表示されます。ただし合計には加算されません。

**Q: 持ち回りのチェックを入れたら何が起きる？**
A: ルート別タブで、その会場のセルに「→」が自動表示されます。年次更新で会場順が入れ替わった場合は、チェックを入れ直してください。

**Q: 品目を登録したのに品目詳細で会場が表示されない**
A: 品目詳細パネルは企画設定の全会場を表示します。期を正しく選択しているか確認してください。

**Q: xlsx取込で0件追加になる**
A: Excelの「貴学名」「配付資料名」が空のシートはスキップされます。また、同名の品目が既に登録されている場合もスキップされます。

**Q: 部数を入力したら自動保存される？**
A: はい。入力欄から離れると自動的に保存されます。保存中は枠がオレンジ色に、保存完了すると緑色に一瞬変わります。

**Q: ルート別タブから印刷したい**
A: ルート別タブを表示した状態でフィルタバー右側の「印刷」ボタンをクリックしてください。タイトルや出力日付付きで印刷できます。

**Q: 配布先明細シートを作り直したい**
A: シートを削除してから、スプレッドシートのメニュー「配付物管理」→「配付物管理シートを作成」を実行してください。
