# アカウント・権限

## このシステムで使っているアカウント

| アカウント | 何に使っているか |
|---|---|
| eibi.soudankai@gmail.com | システムの実行アカウント。メール送信・Webアプリの動作・自動通知など、すべてこのアカウントで動いています |
| soudankai-s@eibi.co.jp | メールの送信元として表示されるアドレス。参加校にはこのアドレスから届いたように見えます |
| eibi.chikoshima@gmail.com | 開発用アカウント。プログラムの編集やGitHub管理に使用 |

## メールの流れ

参加校にメールを送ると、以下の流れで届きます。

```
① システムが eibi.soudankai@gmail.com からメールを送信
  ↓
② 送信元の表示は soudankai-s@eibi.co.jp（参加校にはこちらが見える）
  ↓
③ 参加校のメールアドレスに届く
```

- 送信済みメールは `eibi.soudankai@gmail.com` のGmail「送信済み」フォルダで確認できます
- 参加校が返信すると `soudankai-s@eibi.co.jp` 宛てになりますが、`eibi.soudankai@gmail.com` に転送されるのでGmailで受信できます

## プログラム更新（デプロイ）時の注意

プログラムを更新（デプロイ）するときは、**必ず `eibi.soudankai@gmail.com` のアカウント**で行ってください。

> **なぜ？**
> 別のアカウントでデプロイすると、すべてのWebアプリ（フォーム・ビューア・メール送信画面など）の実行アカウントがそのアカウントに切り替わってしまいます。
> その結果、メールの送信元が変わったり、スプレッドシートへのアクセス権限が変わったりして、正常に動作しなくなる可能性があります。

自動通知のトリガー（CL変更検知）も同じアカウントで設定してください。

## ポータルのパスワード

[ポータル](../portal.html) はパスワードで保護されています。

- パスワードは暗号化（AES）されてHTMLに埋め込まれています
- パスワードを変更する場合は、暗号化データの再生成が必要です（開発者に依頼してください）
