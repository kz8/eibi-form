# 集荷案内・リマインドメール送信

## この機能でできること

### 集荷案内メール（Guidance）
参加校に対して「荷物をこの方法で送ってください」という案内メールを送信します。
メールには参加する会場の一覧・送付期限・荷物登録フォームへのリンクが含まれます。

### リマインドメール（Reminder）
集荷案内を送信したけれどフォーム未回答の学校に対して、督促メールを送信します。
- 送付期限の3日前以内 → 「期限が近づいています」リマインド
- 送付期限を過ぎている → 「期限超過」リマインド
- フォーム回答済みだが倉庫へ送付なのに未入庫 → 「未入庫」リマインド

> **終了済み会場は自動でスキップ**：開催日が前日以前の会場（=既に終了したイベント）は、リマインド対象から外します。集荷案内も同様。「あれ？昨日の会場のリマインドが今頃きた」を防ぐためです。

メールを受け取った参加校は、リンクから [荷物登録フォーム](guide-form.md) にアクセスして送付方法を登録します。

## 画面の開き方

通常はポータルから開きます。

1. [ポータル](../portal.html) にアクセスしてパスワードを入力
2. メニューから以下を選択
   - **集荷案内メール送信** — 通常の集荷案内
   - **リマインドメール送信** — 未回答校への督促

> **別の方法（スプレッドシートから）**: スプレッドシートを開き、メニューから起動することもできます。
> この場合は必ず `eibi.soudankai@gmail.com` のアカウントでスプレッドシートを開いてください。
> 別のアカウントで開くと送信元が変わってしまいます。

## 集荷案内メール 操作手順

### ステップ1: 送信対象を絞り込む

画面上部の「送付期限」欄で、期限の範囲（開始日〜終了日）を指定します。
指定した期間に送付期限があるルートを持つ学校だけが、下の一覧に表示されます。

### ステップ2: 送信先を確認する

一覧に表示された学校にはチェックが入っています（フリガナ50音順）。
送信したくない学校がある場合はチェックを外してください。

> **「送信済みも含める」チェックボックス**:
> デフォルトではOFFになっており、すでに送信済み・下書き作成済みの学校は一覧に表示されません。
> 再送したい場合はこのチェックをONにすると、送信済みの学校も表示されます。

### ステップ2.5: メール本文プレビュー

一覧の **行をクリック** すると、その学校宛のメール本文がポップアップで確認できます。
- 件名・本文HTMLがそのまま表示される
- 宛先・CC・対象ルート数も同時に表示
- ESCキーまたは × ボタンで閉じる

### ステップ3: 送信モードを選んで実行

3つのモードがあります。目的に合わせて選んでください。

| モード | どうなるか | いつ使う？ |
|---|---|---|
| **通常送信** | 参加校にメールがすぐ届きます | 本番送信のとき |
| **下書き作成** | Gmailの「下書き」フォルダに保存されます（まだ届きません） | 送信前に内容を確認したいとき |
| **テスト送信** | 自分のアドレス宛に届きます（参加校には届きません） | メールの見た目を確認したいとき |

モードを選択して「送信」ボタンをクリックすると実行されます。

## リマインドメール 操作手順

### ステップ1: 抽出

「リマインド送信済みも含める」のチェックボックスを必要に応じてON/OFFして、「抽出」ボタンをクリックします。

抽出される対象：
- 集荷案内メール送信済み
- フォーム未回答 OR フォーム回答済みだが倉庫へ送付なのに未入庫
- 送付期限の3日前以内 OR 期限超過

### ステップ2: 送信先を確認する

一覧には対象校が表示されます。各行の「区分」列で：
- **期限前** — まだ期限前だが3日以内
- **超過** — 期限を過ぎている
- **前+超過** — 複数ルートがあり、両方該当

「期限前のみ」「超過のみ」のリンクで一括フィルタも可能です。

集荷案内と同じく **行をクリックでメール本文プレビュー** ができます。

### ステップ3: 送信モードを選んで実行

集荷案内と同じく3モード（通常送信/下書き作成/テスト送信）から選択。

## 自動下書き作成（毎朝7-8時）

集荷案内・リマインドの両方とも、**毎朝自動で下書きが作成される** ようにトリガー設定されています。

### 仕組み
1. 毎朝7-8時にGASトリガーが起動（**平日のみ**、土日はスキップ）
2. 自動で対象抽出
3. Gmailの下書きフォルダに下書きが作成される
4. `chikoshima@eibi.co.jp` 宛に結果通知メールが届く

### 通知メールの例
```
【集荷案内メール 自動下書き作成】

実行日時: 2026/04/08 07:30
対象期限範囲: 2026-04-22 〜 2026-04-29
ステータス: 下書き作成完了
下書き作成: 15 件
エラー: 0 件

Gmailの下書きフォルダで内容を確認の上、送信してください。
```

### 出社後の運用
1. 通知メールで件数把握
2. Gmail下書きフォルダ（https://mail.google.com/mail/u/0/#drafts）を開く
3. 内容確認 → 問題なければ各下書きを送信
4. 不要な下書きは個別に削除可能

### 作成範囲（変更可）
- **集荷案内**: 期限**30〜37日後**の校（締切1ヶ月前に発動。`guidanceConfig.gs` の `autoDraftGuidanceDaysFrom/To` で変更可。2026-04-24に旧14〜21日後から変更）
- **リマインド**: 自動抽出条件と同じ

## 送信後の確認

送信が完了すると、スプレッドシートの「集荷メール」列が自動で更新されます。

| モード | 集荷メール列の表示 |
|---|---|
| 通常送信 | `前期済 MM/DD`（送信日） |
| 下書き作成 | `下書き MM/DD`（作成日） |
| テスト送信 | 変更なし |

送信済みメールの確認は `eibi.soudankai@gmail.com` の Gmail「送信済み」から行えます。

リマインドメールの送信ログは別シート「リマインドログ」に記録されます。

## CC（同報送信）について

メールのCCは以下のルールで決まります。

- **CC希望フラグが「希望」の学校のみ**: その学校の担当社員のメールアドレスがCCに入ります
- CC希望フラグは営業SS（別のスプレッドシート）で管理されています
- CC希望フラグが設定されていない学校には、CCなしで送信されます

> CCの設定を変更したい場合は、営業SSのCC希望フラグを修正してください。

## メールに含まれる内容

### 集荷案内メール
- **参加会場の一覧と送付期限** — ルートごとに会場名と期限が表示されます
- **キャンセル待ちの会場** — キャンセル待ちがある場合は別枠で表示されます
- **倉庫送付先の住所カード（インライン記載）** — 「倉庫へ送付」する場合の宛先住所がメール本文内に表示されます
- **荷物登録フォームへのリンク**
- **📘 進学相談会 実施ガイド緑ボタン** — 当日の運営方法・集合時間等のガイドへのリンク
- **フォームが表示されない場合の注意書き（赤枠）** — 複数Googleアカウント問題への対処法（Chrome / Edge いずれも対象。2026-04-28文言更新）
- **添付ファイル** — 「荷物取扱いのご案内」PDF
- **署名**

### リマインドメール
- **未回答ルート一覧** — 期限超過と期限前を別枠で表示
- **督促文** — 期限状況に応じて文面が変わる
- **「資料はご持参いただく扱い」注意文（黄色枠）** — フォームボタン直前。期限超過時の運用上の前提を明示（2026-04-14追加）
- **荷物登録フォームへのリンク**
- **フォームが表示されない場合の注意書き**
- **📘 進学相談会 実施ガイド緑カード（本文末尾・署名直前）** — 2026-04-24追加。集荷案内とリマインドの導線を統一
- **署名**

## フォームが表示されない問題

メール本文に注意書きを追加していますが、根本対策は技術的に困難です。

### 原因
- Gmail の `saferedirecturl` がリンクをラップする
- **複数の Google アカウントにログインしている環境**では Google が iframe 内 URL を `/u/0/`/`/u/1/` 等のアカウント選択経由に書き換え、iframe 表示が弾かれる
- ブラウザ自体の問題ではなく Google のセッションクッキー挙動が根本。**Chrome / Edge いずれも発生する**（Edge は最近のバージョンでマルチアカウント機能が強化されたため、2026-04-28 に発生確認）

### 対処法
1. **別ブラウザ**で URL を貼り付ける（**Safari / Firefox**）
2. **シークレットモード（プライベートウィンドウ）**で開く
3. 一度すべての **Google アカウントからログアウト**してから開く
4. メール本文のプレーンテキストURL（ボタンの下）を直接コピペする

これらをメール本文の赤枠で案内しています。

> **Edge は推奨しない**：以前は Chrome の代替として Edge を案内していたが、Edge も同症状を示すため **Safari / Firefox** を推奨する文言に変更（2026-04-28）。

## 法人一括フォーム回答の取り扱い

法人で集荷案内を送ると、代表校のメールアドレスに `?corp=法人コード` 付きのリンクが届きます。
法人がフォーム送信すると、`荷物登録_回答` シートには **代表校コード** で記録されます。

リマインドメールと集荷確認タブでは、**代表校コードでの回答を法人内全メンバー校に適用** するロジックで動作しています（4/7修正済み）。これにより、法人で答えた場合にメンバー校が誤って未回答扱いされることはありません。

## よくある質問

**Q. 間違えて送信してしまった場合は？**
→ 送信済みメールの取り消しはできません。訂正が必要な場合は、内容を修正して再度通常送信してください。

**Q. 下書きを作成した後、Gmailから手動で送信した場合は？**
→ 集荷メール列は自動で更新されません。`下書き MM/DD` を `前期済 MM/DD` に手動で書き換えてください。

**Q. 自動下書き作成の下書きを消したらまた作成される？**
→ はい。リマインドの場合、「リマインド送信済みフラグ」は実際に送信した時に立つので、下書きを消して再実行すれば再度作成されます。

**Q. 特定の学校だけ送信対象から外したい場合は？**
→ 2つの方法があります。
- 一時的に外す場合: 一覧のチェックを外す
- 常に外す場合: [参加状況ビューア](guide-opview.md) のダッシュボードで「案内不要フラグ」を設定

**Q. 送信元のアドレスは？**
→ 参加校には `soudankai-s@eibi.co.jp` から届いたように見えます。
返信は `eibi.soudankai@gmail.com` に届きます（転送設定済み）。

**Q. 自動下書き作成は土日も動く？**
→ 動きません。`autoCreateGuidanceDrafts` / `autoCreateReminderDrafts` 内で曜日チェックして土日はスキップしています。

**Q. 1校だけ手動で集荷案内/リマインドを送りたい場合は？**
→ [参加状況ビューア](guide-opview.md) のダッシュボードから可能。学校名クリック → 詳細パネルに4つのボタンが並んでいます：
- **「集荷 下書き」（緑）** — Gmail下書き作成
- **「集荷 即送信」（濃緑）** — その場で送信
- **「ﾘﾏｲﾝﾄﾞ 下書き」（赤）** — Gmail下書き作成
- **「ﾘﾏｲﾝﾄﾞ 即送信」（濃赤）** — その場で送信

リマインド系は事前診断が走り、致命的な不整合（メール未設定・案内不要等）は **block**（送信不可）、警告（全会場終了済み・既送信等）は **warn**（confirmで突破可能）として扱われます。
