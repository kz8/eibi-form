# LuggageWebApp Wiki

荷物登録・集荷管理システムの運用マニュアルです。

## 業務フロー

```
①メール送信        ②フォーム登録       ③入庫記録
運営が参加校へ   →  参加校が荷物情報  →  届いた荷物を
集荷案内を送信      をフォームで登録     入庫処理
```

各ステップの進捗は [参加状況ビューア](guide-opview.md) で一元管理できます。

## 機能一覧

| ステップ | 機能 | 概要 |
|---|---|---|
| ① | [集荷案内メール送信](guide-mail.md) | 参加校への集荷案内メールの作成・送信 |
| ② | [荷物登録フォーム](guide-form.md) | 参加校が荷物の送付方法を登録するフォーム |
| ③ | [入庫記録](guide-receiving.md) | 届いた荷物の入庫処理・記録 |
| 通し | [参加状況ビューア](guide-opview.md) | ダッシュボード・集荷確認・仕分表・フォーム回答 |
| 配付物 | [配付物管理](guide-distribution.md) | 品目登録・会場別部数管理・xlsx取込・ルート別一覧 |

## 横断ガイド（業務目線・年間目線・判断目線）

機能ごとの画面操作を超えて、業務全体を俯瞰したい時のためのページ。

- **[シナリオ別ガイド](scenarios.md)** — 「集荷案内を出すまで」「QRで入庫」「年次更新」など、**業務シーン起点**で関連画面・トリガー・シートを集約
- **[季節カレンダー](calendar.md)** — **月別タイムライン**で「この時期にやること／確認すること／トリガー状態」を整理。年に1度しか発生しないタスク（GitHub Actions Disable 等）の取りこぼし防止
- **[運用判断ログ](decisions.md)** — 「いつ・何を・なぜそう決めたか」の蓄積。3ヶ月後・1年後に「なぜこの設定？」を思い出すための判断記録
