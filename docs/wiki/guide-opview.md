# 参加状況ビューア

## この機能でできること

参加校の進捗状況を一覧で確認・管理するための画面です。
メール送信の状況、フォームの回答状況、荷物の入庫状況など、業務全体の進捗をこの画面で把握できます。

## 画面の開き方

1. [ポータル](../portal.html) にアクセスしてパスワードを入力
2. メニューの「参加状況ビューア」をクリック

## 共通の操作

### フィルタバー

画面上部のフィルタバーで表示する対象を絞り込めます。（集荷確認・仕分表・参加OPタブで共通）

| フィルタ | 操作 |
|---|---|
| **期** | 「前期」「後期」などを選択。選択するとルート一覧が切り替わります |
| **ルート** | 物流ルートを選択。選択すると参加校一覧が表示されます |
| **区分** | 大学・短大・専門学校で絞り込み（すべて / 大学・短大 / 専門学校 / 大学のみ / 短大のみ） |
| **管轄** | 本社のみに絞り込み（第一営業部・第二営業部・社長） |

> 「期」と「ルート」を変更するとサーバーからデータを取得します。「区分」と「管轄」はその場で即座にフィルタされます。

### 印刷

各タブの表示内容をそのまま印刷できます。ブラウザの印刷機能（Ctrl+P）を使ってください。印刷用のレイアウトが自動で適用されます。

## タブの説明

### 集荷確認タブ

会場（ルート）ごとの参加校一覧と、集荷業務の進捗を確認するメインのタブです。

**表示される列：**

| 列 | 内容 |
|---|---|
| 県 | 学校の所在県 |
| 学校名 | 学校名（クリックすると入庫ダイアログが開きます） |
| 会場ステータス | 各会場の参加状況（○/P/Ⓡ/ｷ/●） |
| 予定個口 | フォームで登録された個口数 |
| 入庫個口 | 倉庫に届いた個口数 |
| 集荷メール | メール送信状況（「前期済 MM/DD」「下書き MM/DD」など） |
| フォーム | フォーム回答状況（回答済 / 未回答） |
| 登録時備考 | フォーム送信時に入力された備考 |
| 入庫時備考 | 入庫記録時に入力された備考（読取専用） |
| 仕分表備考 | 仕分けに関するメモ（ここに直接入力できます。入力欄を離れると自動保存） |

**こんなときに使います：**
- 「この会場にどの学校が参加しているか」を確認したい
- 「荷物は届いているか」「フォームは回答済みか」をまとめて確認したい
- 仕分け作業の備考を記録したい

---

### 仕分表タブ

届いた荷物を会場ごとに仕分けるための一覧です。集荷確認タブより表示項目を絞り、仕分け作業に特化しています。

**表示される列：**

| 列 | 内容 |
|---|---|
| 県 | 学校の所在県 |
| 学校名 | 学校名 |
| 会場ステータス | 各会場の参加状況 |
| 個口 | 入庫された個口数 |
| 仕分表備考 | 仕分けに関するメモ（直接入力可。自動保存） |

**こんなときに使います：**
- 倉庫で印刷して、仕分け作業のチェックリストとして使う
- 仕分け時のメモや注意事項を記録する

---

### 参加OPタブ

参加ステータスをマトリクス（表）形式で表示します。

**表示される列：**

| 列 | 内容 |
|---|---|
| 県 | 学校の所在県 |
| 学校名 | 学校名 |
| 会場ステータス | 各会場の参加状況（○/P/Ⓡ/ｷ/●） |

**こんなときに使います：**
- 学校×会場の参加状況を一覧で俯瞰したい
- 特定の会場への参加校を素早く確認したい

---

### リモート参加タブ

リモート参加（Ⓡステータス）の学校を管理する画面です。2つのサブタブがあります。

**参加校一覧サブタブ：**
- Ⓡステータスの学校×会場の一覧を表示
- 期の選択で切り替え

**ブース管理サブタブ：**
- Zoomブースの割当管理とTamaya機材の管理を行います

| 列 | 内容 |
|---|---|
| アカウント | Zoomアカウントのメールアドレス |
| 割当校 | プルダウンから学校を選択（本社管轄が先頭表示） |
| 参加校コード | 割当校に連動して自動セット |
| 割当ルート | 割当校に連動して自動セット |
| 該当会場 | 割当校に連動して自動セット |
| URL連絡 | 未 / 済 / -- から選択 |
| Tamaya機材列 | 見積依頼〜返却の日付、発注No.、機材No. |

---

### 資料仕分けタブ

学校ごとの資料配分を管理し、仕分け指示書を作成するための画面です。

**操作手順：**
1. 学校名を入力（候補が表示されます）
2. 資料名・総数を入力
3. ルート別の配分が会場数に応じて自動計算されます
4. 必要に応じて配分を手動調整
5. 仕分け完了したら個口数を入力して登録

**その他の機能：**
- 登録済みの個口は「N個口 [取消]」と表示され、取消で再入力可能
- 「荷札生成」ボタンで青テーマの荷札PDFをダウンロードできます
- 印刷するとA4サイズの仕分け指示書として使えます

---

### 入庫履歴タブ

[入庫記録](guide-receiving.md) で登録された入庫の履歴を検索・閲覧できます。

**表示される列：**

| 列 | 内容 |
|---|---|
| 学校名 | 学校名 |
| コード | 参加校コード |
| ルート | 荷物ルート |
| 送付方法 | 倉庫へ送付 / 会場直送 / 持込 |
| 予定個口 | フォームで登録された予定数 |
| 入庫日 | 入庫した日付 |
| 入庫個口 | 実際に届いた個口数 |
| 備考 | 備考 |

**操作：**
- 検索欄に学校名・コード・ルートを入力して絞り込めます

---

### フォーム回答タブ

[荷物登録フォーム](guide-form.md) から送信された回答を、学校ごとの最新回答で表示します。

**表示される列：**

| 列 | 内容 |
|---|---|
| 学校名 | 学校名 |
| コード | 参加校コード |
| 回答者 | フォームを送信した人の名前 |
| 回答日時 | 最新の回答日時 |
| ルート | 送付先ルート |
| 送付方法 | 倉庫へ送付 / 会場直送 / 持込 |
| 個口 | 送付予定の個口数 |
| 備考 | 回答時の備考 |

**フィルタ：**
- **期** — 該当期のルートに対する回答のみ表示
- **ルート** — 「すべて」または特定のルートで絞り込み
- **検索** — 学校名・コードで検索

> 同じ学校が複数回フォームを送信した場合、最新の回答のみ表示されます。

---

### ダッシュボードタブ

全校の進捗管理と学校情報の編集ができる、最も多機能なタブです。

**表示される列：**

| 列 | 内容 |
|---|---|
| 県 | 学校の所在県 |
| 学校名 | 学校名（クリックすると編集モーダルが開きます） |
| 区分 | 大学 / 短大 / 専門学校 |
| 担当 | 担当営業名 |
| 管轄 | 管轄部署 |
| 法人名 | 法人に所属している場合の法人名 |
| 集荷メール | メール送信状況 |
| フォーム | 回答済（緑）/ 一部回答（黄）/ 未回答（赤） |
| 入庫 | 入庫個口数 |
| 仕分 | 倉庫仕分の有無 |
| フォームURL | URLコピーボタン |

**フィルタ：**

| フィルタ | 内容 |
|---|---|
| 検索 | 学校名・法人名で検索（Enterで抽出実行） |
| 区分 | 大学 / 短大 / 専門 |
| 管轄 | 管轄部署で絞り込み |
| メール | 送信済 / 未送信 |
| フォーム | 回答済 / 一部回答 / 未回答 |
| 入庫 | 入庫あり / 未入庫 |
| 仕分あり | 倉庫仕分フラグありのみ表示 |
| 案内不要除外 | 案内不要フラグの学校を除外（デフォルトON） |

> 初回は「抽出」ボタンをクリックしてデータを読み込みます。読み込み後はフィルタの切り替えで即座に表示が変わります。

## ダッシュボードの使い方

### 学校情報の編集

学校名をクリックすると編集画面（モーダル）が開きます。以下の情報を編集できます。

| 項目 | 説明 |
|---|---|
| 案内不要フラグ | ONにすると [集荷案内メール](guide-mail.md) の送信対象から外れます |
| 担当営業 | プルダウンから選択（社員マスタから取得） |
| 担当部署 | 学校側の担当部署名 |
| 担当者名 | 学校側の担当者名 |
| Tel | 担当者の電話番号 |
| メール | 担当者のメールアドレス |
| 倉庫仕分 | 倉庫での仕分け区分 |
| 集荷備考 | 集荷に関する備考・特記事項 |

編集が終わったら「保存」ボタンをクリックしてください。

### 参加会場・進捗

モーダル上部には、その学校の参加会場ごとの進捗が表示されます。

| 列 | 内容 |
|---|---|
| 期 | 前期 / 後期 |
| 企画ID | 企画の識別コード |
| 日付 | 開催日 |
| 会場 | 開催都市名 |
| 状況 | 参加ステータス（○/P/Ⓡ/ｷ/●） |
| フォーム | フォーム回答状況と送付方法 |
| 入庫 | 入庫の進捗バー（入庫個口 / 予定個口） |

### フォームURLコピー

モーダル内の「URLをコピー」ボタン、または一覧テーブルの「Copy」ボタンで、その学校の [荷物登録フォーム](guide-form.md) のURLをクリップボードにコピーできます。
参加校にメール以外の方法でフォームURLを伝えたいときに使います。

### 荷札生成

モーダル内の「荷札生成」ボタンで、その学校の荷札PDFを生成・ダウンロードできます。

### 営業SS差異の確認・反映

営業SS（別のスプレッドシート）と学校基本情報に差異がある場合、モーダル内にオレンジの枠で差異が表示されます。

- **「反映」ボタン** — 営業SSの値を学校基本情報に上書きします
- **「確認済み」ボタン** — 差異を認識した上で現在の値を維持します（バッジが消えます）

> 差異がある学校は一覧テーブルの学校名横に赤いバッジが表示されます。変更依頼がある場合は青いバッジが表示されます。

## よくある質問

**Q. 画面の表示が古い情報のままになっている場合は？**
→ データはキャッシュ（一時保存）されているため、最新の情報が反映されるまで数分かかることがあります。
すぐに最新の情報を見たい場合は、スプレッドシートのメニュー「システム」→「キャッシュクリア」を実行してからページを再読み込みしてください。

**Q. 案内不要フラグを設定したのにメールが送られた場合は？**
→ フラグを設定した後にメール送信した場合のみ有効です。すでに送信済みのメールには影響しません。

**Q. 営業SS差異とは何ですか？**
→ 営業部門が管理している別のスプレッドシートと、このシステムの学校情報との間に食い違いがある場合に表示されます。担当部署・担当者名・メールアドレスが対象です。

**Q. フォーム回答が「一部回答」と表示されるのはどういう意味ですか？**
→ その学校が参加する複数のルートのうち、一部のルートのみフォーム回答がある状態です。すべてのルートに回答すると「回答済」になります。

**Q. 法人の学校がグループ化されて表示されるのはなぜですか？**
→ 同じ法人に所属する学校をまとめて表示しています。ただし、法人に所属する学校が1校だけの場合は個校として表示されます。

**Q. ダッシュボードで表示される学校が参加校の全てではないのはなぜですか？**
→ ダッシュボードは縦持ち_自動シートに登録がある学校（＝該当期に参加ステータスがある学校）のみ表示します。学校基本情報に登録があっても参加データがなければ表示されません。
