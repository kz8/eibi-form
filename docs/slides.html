1. 参加状況ビューア

参加校の進捗を一覧で確認・管理する画面

ポータル → 「参加状況ビューア」

共通操作

操作説明
期前期/後期を切替。ルート一覧が変わる
ルート物流ルートを選択 → データ取得
区分大学・短大 / 専門学校 で絞込
管轄本社のみに絞込
A4縦/横印刷・PDFの用紙向き（集荷確認・仕分表のみ）
印刷 / PDF保存印刷ダイアログを開く。PDFは「送信先→PDFに保存」

集荷確認タブ

会場ごとの参加校一覧と集荷業務の進捗を確認するメインのタブ

列内容
会場ステータス○ P Ⓡ ｷ ● で参加状況を表示
予定個口 / 入庫個口フォーム登録数 / 実際に届いた数
集荷メール送信状況（前期済 MM/DD など）
フォーム回答済 / 未回答
仕分表備考直接入力可。改行も可能。自動保存
学校名クリック → 入庫ダイアログが開き、個口数を直接編集できます

仕分表タブ

倉庫での仕分け作業に特化した一覧

  • 表示列：県 / 学校名 / 会場ステータス / 個口 / 仕分表備考
  • 印刷して仕分け作業のチェックリストに使える
  • 仕分表備考は改行入力可・自動保存

参加OPタブ / リモート参加タブ

参加OP

学校×会場の参加ステータスをマトリクス表示

リモート参加

  • 参加校一覧 — Ⓡステータスの学校を一覧
  • ブース管理 — Zoomアカウント割当・Tamaya機材管理・ラベル生成

資料仕分けタブ

学校ごとの資料配分を管理し、仕分け指示書を作成

  • 学校名入力 → 資料名・総数入力 → ルート別自動配分
  • 手動で配分調整可能
  • 仕分け完了 → 個口数入力して登録
  • 荷札生成ボタン / 配布予定表ボタンで PDF ダウンロード
  • 印刷は A4 横で出力

ダッシュボードタブ

全校の進捗管理と学校情報の編集ができる多機能タブ

  • 学校名クリック → 編集モーダル（担当者・Tel・Mail・備考など）
  • フォームURLコピー / 荷札生成
  • 営業SS差異の確認・反映・確認済みマーク
  • 変更依頼の処理（反映 / 却下）
  • 多彩なフィルタ（区分・管轄・メール・フォーム・入庫・仕分あり 等）

クイック検索タブ NEW

電話対応中に1校だけ素早く調べる軽量検索

  • 学校名・コード・フリガナで検索 → 即座に結果表示
  • ダッシュボードのように全校データを読まないので高速（数百ms）

表示内容：

  • 担当者情報（部署・氏名・Tel・Mail）
  • 集荷メール・フォーム回答状態
  • ルート別進捗テーブル（送付方法・個口・入庫）
  • 営業SS差異

入庫履歴 / フォーム回答 / 取込履歴

入庫履歴

入庫記録の検索・閲覧。最大50件、入庫日の新しい順で表示

フォーム回答

荷物登録フォームの回答を学校×ルートで最新表示

取込履歴

CSV自動取込の結果ログ。OK/WARNING/ERRORのステータスとサマリー

2. 配付物管理

資料・備品を一元管理し、会場ごとの部数を決定する画面

ポータル → 「配付物管理」

4つのタブ

タブ用途
品目一覧品目の登録・編集・削除・ドラッグ並べ替え
ルート別ルート×品目の部数を横展開。印刷で仕分け用紙に
会場別 NEW全会場×封入品目のクロス表。俯瞰把握
xlsx取込Excel申込書から品目を一括登録

品目一覧タブ

  • 4セクション：封入 → 単独配布 → 置き → 備品
  • ☰ ドラッグで並べ替え（同セクション内）
  • 行クリック → 下部に品目詳細パネル展開（会場別部数入力）
  • 「+ 品目登録」で新規追加。封入は封入数が自動入力
  • CL名・資料名の変更は全明細に連動

ルート別タブ

  • ルート選択 → 品目×会場の部数一覧
  • 持ち回り会場は「→」で表示
  • 済チェックで作業完了を記録
  • 印刷除外チェック NEW — チェックした行は印刷時に非表示
備品セクションは数字未入力でも常に表示されます。「データありのみ」チェックで切替可能。

会場別タブ NEW

期全体の全会場×封入品目のクロス表

  • 行：全企画（管轄順 → ルート → 日付順）
  • 列：封入品目（栄美通信の社内品は除外）
  • セル：部数あり＝緑 / なし＝空白
  • 管轄グループ（東京→札幌→仙台→大阪→福岡）で区切り
  • サマリー：企画数・封入品目数・合計部数

xlsx取込タブ

  • 顧客の配付物申込書（Excel）をアップロード
  • 各シート＝1品目。会場名・開催日で企画設定と照合
  • マッチした会場に倉庫部数を自動入力
  • 既存の同名品目はスキップ
  • 未マッチの都市がある場合は結果メッセージに表示

3. 荷物登録フォーム

参加校が送付方法と個口数を登録するWebフォーム

集荷案内メール内のリンクからアクセス

参加校の操作フロー

  1. フォームを開く — メールのリンクをクリック
  2. 送付方法を選ぶ — 倉庫へ送付 / 会場直送 / 持込
  3. 個口数・備考を入力
  4. 送信 → 確認メールが届く
同じURLから何度でも再登録可能（最新の内容で上書き）

送付方法ごとのご案内カード NEW

「倉庫へ送付」を選択すると：

  • 倉庫の住所・宛名・電話番号が表示される
  • 「住所をコピー」ボタンで送り状記入に使える

「会場直送」を選択すると：

  • 着日ルール（当日午前中着 / 一部前日着）が表示される
  • 荷物取扱いのご案内へのリンク

荷札PDF / リンクボタン

荷札PDF

  • 通常モード（ルートごとに1枚）/ 会場別モード
  • QRコードの下に「テープを貼らないで」の注意書きあり

フォーム画面のリンクボタン

  • 📦 荷物取扱いのご案内 — 梱包・発送・返送の手順
  • 📘 進学相談会 実施ガイド — 当日の運営方法・集合時間

4. 入庫記録

届いた荷物の入庫処理を行う画面

ポータル → 「入庫記録」

操作手順

  1. 学校を検索 — 学校名またはコードを入力 → 候補から選択
  2. ルートを選択 — 参加しているルート一覧から該当を選択
  3. 入庫個口数を入力 — 予定個口数と照合しながら入力
  4. 保存

履歴タブ

直近50件の入庫履歴を入庫日の新しい順に表示

ビューアの集荷確認タブからも入庫個口を直接編集できます

5. 集荷案内・リマインドメール

参加校への案内メール送信・督促メール送信

ポータル → 「集荷案内メール送信」/「リマインドメール送信」

集荷案内メール — 操作手順

  1. 送付期限の範囲を指定 → 対象校が一覧表示
  2. 不要な学校のチェックを外す
  3. 行クリックでメール本文プレビュー
  4. 送信モードを選んで実行
モード動作
通常送信参加校にメールが届く
下書き作成Gmailの下書きに保存（まだ届かない）
テスト送信自分宛に届く（参加校には届かない）

メールに含まれる内容

  • 参加会場一覧と送付期限
  • キャンセル待ち会場（該当あれば）
  • 倉庫送付先の住所（本文に直接記載）NEW
  • 実施ガイドへの誘導カード NEW
  • 荷物登録フォームへのリンク
  • Chrome複数アカウント時の注意書き
  • 添付：荷物取扱いのご案内PDF

リマインドメール

  • 集荷案内送信済みでフォーム未回答 or 未入庫の学校が対象
  • 期限前（3日以内）と期限超過で文面が変わる
  • 操作は集荷案内と同じ3モード

自動下書き作成（毎朝 平日7-8時）

  • 集荷案内・リマインドとも自動で下書きが作成される
  • 通知メールが届く → Gmailの下書きを確認 → 送信

パフォーマンス改善 NEW

ビューア・フォームの表示速度が大幅に向上しました

  • シートデータのキャッシュ導入（2回目以降の表示が高速化）
  • キャッシュサイズ制限（100KB）を突破するチャンク分割対応
  • データ書込み時はキャッシュが自動無効化（データ齟齬なし）
  • フォームのプリフィル（前回回答の自動入力）もキャッシュ対応
表示がおかしい場合は、スプレッドシートのメニュー「システム → キャッシュクリア」を実行してください
